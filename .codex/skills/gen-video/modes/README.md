# Modes

`modes/` 定义 skill 该接管多少。

这层很关键，因为真正的问题不是“能不能写更复杂”，而是：

> 当前模型 / 平台已经会多少？skill 还应该介入多少？

mode 判断默认基于 `profiles/`；如果模型能力近期变化很快，再补看 `../knowledge/` 里的最新记录。

## 三种模式

- `native`
  模型原生做大部分创作，skill 只补目标、约束和交付整理

- `hybrid`
  模型先出主结果，skill 补连续性、真实性、执行结构和整理

- `manual`
  skill 进行高控制拆解，手工明确镜头、资产和分流规则

## 默认倾向

- 如果模型原生能力已经明显强，优先 `native`
- 如果模型原生强，但用户还需要统一交付和稳定约束，优先 `hybrid`
- 只有在明确需要强控制时，才走 `manual`
