# Output Contract

`gen-video` 的核心不是某一套固定 prompt，而是稳定的交付契约。

无论最终走 `native`、`hybrid` 还是 `manual`，最后都应尽量交付下面这些内容中的一部分或全部：

## 1. 故事层

- 一句话故事
- 核心冲突
- 情绪弧线
- 关键转折
- 结尾落点

## 2. 视觉层

- 角色锚点
- 场景锚点
- 道具锚点
- 风格方向
- 连续性约束

## 3. 生产层

- 图片生成材料
- 视频生成材料
- 平台执行建议
- 回灌 / 重做点
- 拼接建议

## 4. 质量层

- 真实性是否成立
- 连续性是否稳定
- 节奏是否匹配时长
- 高潮是否从真实动作或真实物件长出来
- 结尾是否落在最该落的动作上

## 原则

输出契约优先于生成路径。

也就是说，用户真正要的是：

- 更稳定的故事结构
- 更可信的角色与场景
- 更能执行的制作资料

至于这些内容是：

- 模型原生生成
- 模型初稿 + skill 整理
- skill 手工拆解

不是第一优先级。
