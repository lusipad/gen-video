# Core

`core/` 只放 `稳定、不应频繁随模型迭代变化` 的规则。

这里不接收未经验证的新鲜结论。新信息应先经过 `../knowledge/raw/`、`../knowledge/wiki/` 和必要的 benchmark，再决定是否晋升到这里。

这里的内容不回答“某个模型现在强不强”，而回答：

- 什么是一个合格的交付物
- 真实题材哪些约束不能丢
- 情绪片和剧情片哪些结构规律是稳定的
- 无论走哪条生成路径，最后都要满足什么质量门槛

## 应该放在这里的内容

- 输出契约
- 真实性锚点规则
- 叙事结构规则
- 质量检查表
- 用户意图澄清规则

## 不应该放在这里的内容

- 某个模型最近更新了什么能力
- 某个平台 UI 怎么点
- 某个工作流是否临时支持某个画幅
- 某个模型最近更擅长中英文哪一种提示词

这些都应该放到 `profiles/`。
