# Google Flow 路线补充说明

当用户明确说会在 `Google Flow` 中执行，而不是只要一组模型提示词时，使用这份补充说明。

## 1. 先确认你交付的不是“提示词清单”，而是“操作包”

Flow 的最佳交付形态通常不是单条万能提示词，而是：

1. 项目级设置
2. 素材图生成顺序
3. 按镜头拆分的视频提示词
4. 关键帧存帧点
5. 回灌重做点
6. Scenebuilder 拼接顺序
7. 后期补配建议

如果用户要求“从生图到成片完整流程”，要按上面七层交付，而不是只给剧情和分镜。

## 2. Flow 输出要按镜头思考，不按“逐帧控制”思考

- 不要承诺逐帧动画控制。
- 默认按镜头或片段设计，每条镜头单独规划。
- 对长片，优先先搭骨架镜头，再用插入镜头和重做镜头补节奏。

## 3. 常见 Flow 工作流映射

- `Text to Video`：没有可靠参考图，先做空间建立、环境镜头或过渡镜头。
- `Ingredients to Video`：角色、场景、道具一致性最重要时优先。
- `Frames to Video`：要抓关键构图、关键情绪、关键动作起点时使用。
- `Extend`：对单条已满意镜头延长节奏或补呼吸，不要乱接语义完全变化的镜头。

给用户的镜头表里，最好显式标明每条镜头推荐使用哪一种工作流。

## 4. Flow 交付中必须额外写出来的东西

除了普通分镜包，Flow 路线还应额外写：

- 哪些图先用 Nano Banana 2 生成
- 哪些视频先用 Fast 跑通，再用更高质量重做
- 哪些镜头生成后要 `Save frame as image`
- 哪些镜头应用存下来的帧回灌
- 哪些镜头是插入镜头，用来遮切点、补呼吸、拉总时长
- 最终在 Scenebuilder 里的拼接顺序

## 5. 中文提示词是可接受的

如果用户明确会在 Flow 界面里手动复制粘贴，且偏好中文，给 `全中文 prompt` 没问题。  
这时更重要的是：

- 句子清楚
- 主体明确
- 真实约束充分
- 不把静态描述和时间轴描述混写

不要为了“模型更懂英文”而牺牲用户的可操作性，尤其在需要大量手动编辑时。

## 6. 真实题材在 Flow 中更容易跑偏，要先写死锚点

如果项目基于真实地点、地方风貌、非遗、真实职业或真实年龄段角色：

- 每条镜头都重复关键锚点，不要只在第一条里写一次。
- 重点重复 `地点`、`人物年龄/气质`、`道具结构`、`不允许变成什么`。
- 奇观镜头也要从真实物件长出来，不要突然进入无依据的大特效。

## 7. 催泪片的 Flow 规划规则

对催泪现实题材，优先使用这条情绪设计：

`小失败 -> 忍着 -> 偷看/偷学 -> 冲突 -> 点亮真实物件 -> 奇观爆发 -> 极小动作收尾`

注意：

- 高潮时长通常不要长过铺垫时长。
- 最后一个镜头最好回到手、眼神、呼吸、真实道具，而不是停在大场面。
- 越是大场面，收尾越要静。

## 8. 儿童角色与音频

如果主角中有儿童：

- 不要把关键情绪依赖在平台内的生成对白上。
- 更稳的做法是：Flow 里保留环境音、动作音、氛围音乐；对白和旁白走后期。
- 如果一定要在提示词里写声音，也要保证即使语音缺失，情绪仍然成立。

## 9. 默认产物建议

当用户明确说“我要在 Flow 中直接做”时，建议至少输出：

1. `项目参数`
2. `素材图提示词`
3. `镜头表`
4. `关键帧/回灌表`
5. `Scenebuilder 拼接建议`
6. `后期音频建议`
