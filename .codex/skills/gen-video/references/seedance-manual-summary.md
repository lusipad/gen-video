# Seedance 手册工作摘要

这份摘要用于快速执行，不替代原始手册。目标是让你在不通读全文的情况下，抓住最关键的限制、语法和输出规则。

## 1. 平台能力与限制

### 支持的素材类型

- 图片：最多 9 张
- 视频：最多 3 个，总时长不超过 15 秒
- 音频：最多 3 个，总时长不超过 15 秒
- 文本：自然语言描述

### 总体限制

- 总文件数最多 12 个
- 视频与音频参考更耗额度
- 暂不支持写实真人脸部素材

### 常见用途

- 参考图生成视频
- 参考视频复刻运镜或动作
- 对已有视频做续拍
- 对已有视频做剧情编辑或颠覆

## 2. 推荐提问顺序

如果用户明确要 Seedance 风格输出，优先补齐这几类信息：

1. 故事在讲什么
2. 时长是几秒
3. 画幅比例是什么
4. 有没有图片、视频、音频参考
5. 风格、镜头、声音、节奏有没有明确要求

最短提问版本：

```text
我先按 Seedance 分镜来写。请补这 4 个信息：
1. 你想讲什么故事
2. 要几秒
3. 横屏、竖屏还是电影宽屏
4. 有没有参考图、参考视频、参考音频
```

## 3. 标准结构

Seedance 提示词最稳的写法是固定四段：

```markdown
【风格】[风格]，[时长]，[比例]，[氛围]

【时间轴】
0-X秒：[镜头] + [画面] + [动作] + [特效]
X-Y秒：[镜头] + [画面] + [动作] + [特效]

【声音】[配乐] + [音效] + [对白]

【参考】@图片1 [用途]，@视频1 [用途]，@音频1 [用途]
```

## 4. 时间轴的推荐分法

15 秒常用切法：

- 0-3 秒：建立场景
- 3-6 秒：引入主体或冲突
- 6-10 秒：高潮动作或情绪爆发
- 10-13 秒：转折或过渡
- 13-15 秒：结尾或落版

如果只有 8 到 10 秒，就压缩成 3 到 4 段，但仍然保留：

- 建立
- 发展
- 高潮
- 结尾

## 5. 常用引用语法

直接使用 `@素材名` 指定用途：

```markdown
@图片1 作为首帧
@图片2 作为尾帧
@图片3 作为角色形象参考
@图片4-6 作为场景参考
@视频1 参考运镜方式
@视频2 参考动作节奏
@音频1 用于配乐
@音频2 用于对白参考
```

## 6. Seedance 中最重要的 5 类场景

### 角色一致性

- 让用户提供角色参考图
- 在提示词里明确 `@图片1 作为角色形象参考`
- 避免写实真人脸素材

### 运镜或动作复刻

- 让用户提供参考视频
- 明确写 `@视频1 参考运镜和动作节奏`

### 视频延长

- 明确写 `将@视频1延长X秒`
- 指的是新增时长，不是总时长

### 剧情颠覆或视频编辑

- 明确写保留什么
- 明确写修改什么
- 明确写新增什么

### 音乐卡点

- 提供节奏参考更稳
- 时间轴尽量对齐音乐重拍

## 7. 镜头语言速查

| 用途 | 写法 |
| --- | --- |
| 推近主体 | 推镜头 / 缓慢推近 / 急速推进 |
| 拉远场景 | 拉镜头 / 逐渐拉远 |
| 左右展示 | 左摇 / 右摇 / 横移 |

## 9. 最稳的输出格式

最终给用户的 Seedance 输出，尽量固定为四段：

1. 理解确认
2. 可直接复制的完整分镜提示词
3. 建议上传的素材
4. 使用提醒

推荐格式：

```markdown
## 理解确认
[一句话确认故事理解]

## 分镜提示词
[完整 Seedance 提示词]

## 素材建议
- [建议上传什么图]
- [建议上传什么视频]
- [建议上传什么音频]

## 使用提醒
- [如何使用 @素材]
- [时长和比例注意事项]
```

## 10. 执行检查表

在交付前，确认：

- 时间轴是否完整
- 镜头语言是否明确
- 动作描述是否具体
- `@素材` 引用是否清楚
- 声音部分是否写全
- 参考素材用途是否标清

如果这 6 项都满足，Seedance 版本通常已经达到可用线。
