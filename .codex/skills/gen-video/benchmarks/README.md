# Benchmarks

`benchmarks/` 用来避免这个 skill 变成静态经验包。

目标不是追求“规则越来越多”，而是持续比较：

- `native` 是否已经比 `manual` 更好
- `hybrid` 是否比纯手工更轻更稳
- 某个平台升级后，哪些旧规则已经可以退役

benchmark 不是终点。结果要回写 `../knowledge/log.md`，并决定是否更新 `../profiles/` 或退役旧规则。

## benchmark 的最小结构

每一条 benchmark 至少记录：

- 任务类型
- 模型 / 平台
- 使用模式
- 产出质量
- 失败点
- 是否应更新 skill

## 建议覆盖的样题

- 现实题材
- 催泪亲情片
- 非遗 / 地方文化
- 广告短片
- `Google Flow` 中长片
- `Seedance` 自动分镜
