# Veo + Nano Banana Profile

## 角色

这组模型更适合：

- `Nano Banana` 负责静态图、关键帧、角色图、场景图
- `Veo` 负责镜头、动作、时间推进和声音块

## 原生强项

- 图像与视频分工明确
- 关键帧驱动视频
- 角色、场景、道具的参考图工作流

## skill 应怎么介入

- `通常走 hybrid`
- skill 负责：
  - 故事结构
  - 资产表
  - 图片与视频 prompt 分流
  - 关键帧与镜头的衔接

## 什么时候走 manual

- 用户明确需要高控制度
- 连续性要求非常高
- 平台侧没有更高层编排工具时

## 什么时候别写太重

- 如果当前任务只是标准短片，不要把简单镜头拆成过度复杂的 prompt 模板
- 如果 Nano Banana 已经能稳定完成角色 / 场景图，不要重复写一大堆不必要的附加约束

## 高变动点

- 画幅支持
- 图像输入和回灌方式
- 音频表现稳定性
- SDK / 平台能力差异
