# Profiles

`profiles/` 放 `会变化、需要按模型和平台分别判断` 的能力档案。

它不是原始资料库。变化快的依据应先进入 `../knowledge/raw/`，经 `../knowledge/wiki/` 综合后，再编译进这里。

它们的职责不是教你“怎么写得更复杂”，而是帮助 skill 判断：

- 这个模型 / 平台原生已经擅长什么
- 哪些事情应交给模型原生完成
- 哪些事情仍然需要 skill 补约束
- 什么时候应该改走 `native`、`hybrid` 或 `manual`

## 使用方式

1. 先确认模型栈和平台
2. 读取对应 profile
3. 判断当前任务更适合哪种 mode
4. 再决定 skill 要接管多少

如果 profile 与最新官方能力、实测结果或用户实际体验冲突，先去 `../knowledge/log.md` 查最近变更，再决定是否刷新 profile。

## profile 应包含

- 适合处理的任务
- 原生强项
- 容易跑偏的地方
- skill 该退后还是该接管
- 需要额外查官方文档的高变动点
