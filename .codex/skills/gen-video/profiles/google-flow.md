# Google Flow Profile

## 角色

`Google Flow` 是执行平台，不是单一模型。

它更像一个镜头生产与编排界面，适合：

- 素材图管理
- 镜头分段生成
- 关键帧回灌
- 片段延展
- 场景拼接

## 原生强项

- 按镜头组织长片或中片
- 把素材图、视频片段和拼接流程放进同一工作流
- 支持 `Text`、`Frames`、`Ingredients`、`Extend` 这类多路径执行

## 容易跑偏的点

- 用户把它误当成“逐帧控制工具”
- 一上来就想用一条 prompt 生成整片
- 平台能力变化较快，某些工作流支持范围会调整

## skill 应怎么介入

- `优先 hybrid`
- 交给 Flow 原生去做镜头生产和编排
- skill 重点补：
  - 剧情和镜头层级
  - 真实性锚点
  - 回灌点
  - 拼接顺序
  - 音频后期策略

## 何时不要过度介入

- 当 Flow 当前版本已经能稳定完成镜头级规划时，不要再手工把每个镜头拆得过死
- 当用户主要需要“执行顺序”而不是“镜头创作”，skill 不应重复输出过度细碎的美术描述

## 高变动点

- 画幅与工作流支持矩阵
- `Extend` 限制
- `Ingredients` 与 `Frames` 的可用范围
- 平台内音频行为

这些点在正式执行前应优先核对官方文档。
